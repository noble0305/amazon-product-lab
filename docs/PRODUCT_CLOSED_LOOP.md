# 产品方案闭环规划

## 核心对象

系统以“产品方案”作为闭环主对象，而不是直接复制某个对标 ASIN。

一个产品方案包含：

- 多个对标 ASIN 及可借鉴点
- 目标用户、核心痛点和差异化假设
- 多个 1688 供应商与报价版本
- 包装尺寸、重量、合规和知识产权审核
- 三情景利润测算
- Listing 文案和图片素材版本
- 采购、打样、上架及销售复盘状态

## 状态流转

```text
idea
→ benchmarked
→ sourcing
→ quoted
→ sampled
→ approved
→ listing_ready
→ launch_ready
→ launched
→ reviewing
→ scale / stop
```

每次状态变化都记录操作者、时间、证据和理由。高风险、低利润或人工拒绝可以进入 `rejected`，但不得删除历史记录。

## 阶段一：产品方案与对标 ASIN

### 功能

- 从 ASIN 产品机会榜勾选一个或多个对标商品。
- 创建产品方案，填写目标用户、痛点、差异化方向和目标售价。
- 保存对标快照，防止后续竞品页面变化导致证据丢失。
- 标记可借鉴属性与禁止复制内容，避免直接复用商标、图片和受保护文案。

### 验收

- 一个方案可绑定多个 ASIN。
- 每个 ASIN 保存导入批次、原始指标和选择理由。
- 未填写差异化假设的方案不能进入供应商搜索。

## 阶段二：供应商搜寻与报价

### 功能

- 保存 1688 搜索词、商品链接、供应商和工厂信息。
- 记录 MOQ、阶梯价、打样费、交期、材质、定制能力和证书。
- 支持同一供应商多轮报价，保留历史版本。
- 将采购价、包装、头程、关税等转换为到岸成本。
- 上传供应商原图、规格表和聊天确认截图。

### 自动化边界

- 系统可生成 1688 搜索词、询价问题和供应商对比表。
- 不自动下单、付款或向供应商发送消息。
- 供应商自述的证书和工厂能力必须人工核验。

## 阶段三：利润与风险闸门

### 功能

- 录入包装尺寸、重量、采购价、头程、关税、FBA、佣金、仓储、CPC、转化率和退货损失。
- 复用乐观、基准、悲观三情景利润模型。
- 输出盈亏平衡售价、最大可接受采购价和广告成本上限。
- 执行合规、侵权、危险品和 15% 基准利润率红线。

### 验收

- 缺失必填成本字段时禁止进入打样批准。
- 每次报价变化自动生成新的利润快照，不覆盖旧结果。
- 红线未解除时不能进入 `approved`。

## 阶段四：样品与采购决策

### 功能

- 建立样品检查表：尺寸、材质、功能、气味、包装、跌落、标签和说明书。
- 保存样品图片、视频、缺陷和供应商整改记录。
- 对比样品与对标 ASIN 的差异化承诺是否成立。
- 记录采购数量、单价、交期、付款节点和止损条件。

### 自动化边界

- 系统生成检查清单和采购建议。
- 人工确认样品、合同、付款和正式采购。

## 阶段五：Listing 工作台

### 功能

- 基于产品方案、搜索词证据和确认后的规格生成标题、五点、描述和后台搜索词草稿。
- 建立“事实来源表”，每条卖点关联供应商规格、测试或样品证据。
- 检查竞品商标、禁限词、医疗功效承诺和未经证明的绝对化表达。
- 保存多版文案和人工审核状态。

### 验收

- 没有证据来源的产品声明不得进入最终稿。
- 不复制对标 ASIN 文案。
- 最终 Listing 必须人工批准。

## 阶段六：1688 图片处理

### 功能

- 建立素材库，区分供应商原图、样品实拍、设计稿和最终图。
- 记录每张素材的授权来源与可使用范围。
- 支持抠图、背景替换、尺寸规范、排版、信息图和场景图草稿。
- 图片中的尺寸、材质和功能声明与事实来源表联动。
- 输出 Amazon 主图、附图和 A+ 素材包。

### 自动化边界

- 供应商图片未获得授权或包含第三方品牌时不得发布。
- AI 生成场景不得伪造产品结构、配件或效果。
- 最终图片由人工核对实物一致性。

## 阶段七：上架交接与复盘

### 功能

- 生成上架交接包：SKU、类目、价格、文案、图片、合规文件、库存和广告预算。
- 初期只导出清单或模板，不自动调用 Seller Central 发布。
- 上架后记录自然订单、广告订单、CPC、转化率、退货、贡献利润和周转。
- 对比预测与真实结果，区分选品、产品、Listing、价格和广告问题。

## 数据模型建议

- `product_concepts`：产品方案与状态
- `concept_benchmarks`：方案和对标 ASIN 的关联
- `suppliers`：供应商主体
- `supplier_quotes`：报价版本
- `profit_snapshots`：利润测算快照
- `samples`：样品和检查结果
- `evidence_items`：卖点事实来源
- `listing_versions`：Listing 版本
- `media_assets`：图片来源、授权和处理状态
- `launch_packages`：上架交接包
- `experiment_results`：真实销售结果
- `decision_events`：状态变化及理由

首期可使用 Python 标准库 `sqlite3`，无需引入第三方依赖。

## 推荐开发顺序

1. 产品方案、对标 ASIN 选择和状态记录。
2. 供应商与多轮报价管理。
3. 利润快照、最大采购价和风险闸门。
4. 样品检查与采购决策。
5. Listing 事实来源和文案版本。
6. 图片素材来源与处理流程。
7. 上架交接包和销售结果回传。

每阶段都应先完成数据留痕和人工审核，再考虑外部平台自动化。

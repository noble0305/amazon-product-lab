# 闭环 MVP 规格

## 目标

让单个产品方案从对标 ASIN 选择开始，经过供应商报价、利润评估、风险审核、Listing 准备、上架交接和真实结果复盘，形成可追溯闭环。

## 范围

- SQLite 本地持久化分析批次和产品方案。
- 从 ASIN 分析结果选择多个对标商品创建方案。
- 保存目标用户、痛点、差异化假设和目标售价。
- 记录供应商报价、成本、广告和退货假设。
- 自动计算乐观、基准和悲观贡献利润。
- 合规、侵权、危险品和基准利润率低于 15% 时阻止批准。
- 保存 Listing 草稿及事实依据。
- 导出 JSON 上架交接包。
- 保存真实销售结果并计算实际贡献利润。

## 不做

- 不自动抓取亚马逊或 1688。
- 不自动联系供应商、下单或付款。
- 不自动发布 Listing、广告或价格。
- 不自动复用竞品文案和图片。

## 状态

`idea → sourcing → quoted → approved → listing_ready → launch_ready → launched → reviewing → scale 或 stop`

`approved` 前必须满足：成本字段完整、风险红线通过、基准贡献利润率不低于 15%。

`launch_ready` 前必须满足：Listing 标题、五点、描述、搜索词和事实依据已填写，并由人工确认。

## 数据位置

默认数据库：`data/product_lab.db`。数据库和上传业务数据不进入 Git。

## 验收标准

- 页面刷新后自动恢复最近一次分析。
- 可勾选 ASIN 创建产品方案，且对标快照永久保留。
- 修改成本后生成新的利润快照，不覆盖旧记录。
- 红线失败时无法将方案标记为批准。
- 可导出包含 SKU、价格、文案、成本、风险和库存信息的上架交接包。
- 可录入真实结果并显示预测利润与实际利润差异。
- 所有功能仅使用 Python 标准库和浏览器原生 API。
